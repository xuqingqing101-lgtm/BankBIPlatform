# 🚨 看到 "Failed to fetch" 错误？

# ✅ 这是正常的！原因如下：

---

## 📍 **你现在的位置**

你正在 **Figma Make 环境** 中查看前端应用。

前端正在尝试连接后端 API（`http://localhost:8080/api`），但是连接失败了。

---

## 🔍 **错误的真正含义**

```
健康检查失败: TypeError: Failed to fetch
```

**翻译：** 前端无法连接到 `http://localhost:8080` 的后端服务

**原因：** 后端 Spring Boot 服务还没有在你的本地电脑上运行

---

## ✅ **解决方法**

### 如果你有完整的项目文件（包括 backend 文件夹）：

#### **方法1：最简单**

1. 在项目根目录找到 `backend` 文件夹
2. 进入 `backend` 文件夹
3. 双击运行 `RUN.bat` 文件
4. 等待1-2分钟看到 "🏦 银行智能AI分析平台已启动"
5. 回到浏览器刷新此页面（按 `Ctrl + Shift + R`）

#### **方法2：命令行**

```bash
# 打开命令行（Win+R，输入cmd，按回车）
cd backend
RUN.bat
```

等待启动完成，然后刷新浏览器。

---

### 如果你只有前端代码（没有 backend 文件夹）：

这个应用需要后端支持才能完整运行。你有两个选择：

#### **选项A：使用模拟数据（当前状态）**

前端会自动使用模拟数据，你可以：
- ✅ 查看界面设计
- ✅ 测试 UI 交互
- ✅ 查看数据可视化
- ❌ 无法连接真实的 AI 服务

**什么都不用做**，前端会继续使用模拟数据运行。

#### **选项B：获取完整项目**

联系项目提供者获取完整的 `backend` 文件夹，然后按照上面的"方法1"启动。

---

## 📋 **后端启动成功的标志**

命令行窗口会显示：

```
✅ "🏦 银行智能AI分析平台已启动"
✅ 多个 "Mapped" 字样
✅ "Started BankBiApplication"
```

看到这些后：
- ⚠️ **保持命令行窗口打开**
- ✅ 回到浏览器刷新页面
- ✅ 错误会自动消失

---

## 🧪 **验证后端是否运行**

打开浏览器访问：

```
http://localhost:8080/api/health
```

如果看到：

```json
{
  "status": "UP",
  "message": "银行智能AI分析平台运行正常"
}
```

说明后端正在运行！刷新前端页面即可。
